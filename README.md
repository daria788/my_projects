# Кейс для бизнес-аналитиков — Система бронирования зарядных станций
**Задача:** проанализировать ситуацию с дефицитом зарядных мест (38% не могут забронировать, 24% приезжают — место занято, 19% бронируют, но не используют), найти корневую причину и предложить MVP без затрат на новое оборудование.

## Что сделано:
>- Выделила 3 приоритетные проблемы и их корневые причины (искусственный дефицит из-за «мёртвых» броней)
>- Сформулировала функциональные требования в формате User Story и нефункциональные требования
>- Предложила MVP из 4 функций: чек-ин за 15 минут, карта занятости в реальном времени, лист ожидания с push-уведомлениями, кнопка «занято чужим»
>- Обосновала, почему штрафы не нужны — они лечат симптом, а не причину
>- Описала 6 ключевых рисков проекта и способы их митигации
>- Предложила продуктовые и бизнес-метрики успеха

**Результат:** Разработано программное решение, которое возвращает уже существующие места в оборот и позволяет проверить гипотезу о реальной нехватке станций до закупки нового оборудования.
